## Codec preference order

record me follows this order, checking `MediaRecorder.isTypeSupported()` at runtime:

1. **MP4 · H.264 (AAC)** — preferred. Universally playable, best compatibility with media players, editors, and sharing platforms.
2. **WebM · VP9** — fallback. Used when H.264 encoding is unavailable (Firefox, some mobile browsers).
3. **WebM · VP8** — last resort. Older fallback for very limited environments.

This order is contractual — VP9 is never preferred over H.264. The filename in the downloaded file reflects the actual container format (`recording.mp4` or `recording.webm`).

## Why MP4 first?

MP4 (H.264 + AAC) is natively supported by every major platform: macOS QuickTime, Windows Media Player, iOS, Android, Slack, Google Drive, Notion, and virtually every video editor. WebM has excellent quality but requires additional codec installation on some Windows machines and is not natively supported in iOS Safari.

## Container vs codec

| Container | Video codec | Audio codec | Typical size    |
| --------- | ----------- | ----------- | --------------- |
| `.mp4`    | H.264       | AAC         | Smallest        |
| `.webm`   | VP9         | Opus        | Small           |
| `.webm`   | VP8         | Opus        | Slightly larger |

## Checking your browser's support

Open your browser's developer console and run:

```js
MediaRecorder.isTypeSupported('video/mp4;codecs=h264,aac');
MediaRecorder.isTypeSupported('video/webm;codecs=vp9');
```

`true` means the codec is available. Chrome and Edge return `true` for H.264; Firefox returns `false` and falls back to VP9.
